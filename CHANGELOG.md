# Changelog · ai-10x-learning

记录本 skill 的能力演进与借鉴来源。方法论同源（斯坦福 STORM 多视角研究法 + Rahul 六角色学习框架），以下为工程化与跨工具通用化过程中新增的能力。

## 借鉴自 Daliu-Awesome-Skills（大刘开源十步学习法）的特性

本 skill 在保留自有「活导师」交互内核（真人被考的考到崩溃 / 费曼实时循环）的前提下，从大刘版借鉴并落地了以下 5 项：

1. **HTML 成果卡交付层（步骤 10.5，可选）**
   - `assets/template.html`：单文件骨架，7 个占位符 `{{TITLE}}{{ROLE}}{{BUDGET}}{{GOAL}}{{LADDER}}{{FLASHCARDS}}{{QUESTION_BANK}}` 填空。
   - `references/design.md`：自有中性配色规范（蓝 / 青 / 琥珀），明确不沿用第三方品牌色、避免授权争议。
   - 触发条件：只有用户说「导出 HTML / 生成学习卡」才生成；默认仍是 Markdown。
   - 支持局部导出（「只导出速查表」/「只导出题库」仅渲染对应段）。
2. **能力感知联网取证（硬规则 5）**
   - 不写死具体工具名；先判断环境能否实时联网，能则标源，不能则每条标 `[需核实]`，禁止伪造来源 / URL。
3. **逐环交接 checklist（硬规则 7）**
   - 「逐环交接 · 依赖与完成判定」表，标注每步输入 / 产出 / 完成判定；铁律：严禁跳序、严禁凭空生成上游未支撑内容、上游缺失先确认不编造。
4. **开场学习画像 + 主题对齐（第 0 步）**
   - 先复述对主题的理解（范围 / 焦点 / 起点假设 / 不含 / 预期产出形态）确认，再问 3 必答 + 可选补充，防 AI 理解偏航。
5. **反触发护栏**
   - 事实单概念速答 → 直答 / 单步；文档 / PPT → 交别 skill；点名单步 → 只跑那步。

附带：一键安装脚本 `install.sh`（跨 WorkBuddy / Claude Code / Codex / Cursor + 项目级）。

## 本 skill 独有（未借鉴）

- **活导师交互内核**：考到崩溃（考官一次一题、评分 + 差距 + 大白话重讲）、费曼循环（先复述再纠偏）为实时真人参与，测试效应真正发生。
- **三模式路由**：研究(1–4) / 学习(5–10) / 全闭环(1–10)。
- **规范 MIT LICENSE** + 纯对话工具兜底（复制 `references/prompts.md` 全文即可在 ChatGPT / 豆包 / DeepSeek 等使用）。

## 校验与样例

- `scripts/validate_card.py`：导出 HTML 卡后的自检脚本（残留占位符 / DOCTYPE / 闭合标签 / 体积；退出码 0=通过 1=失败）。
- `examples/FDE_通用版_学习卡.html`、`examples/FDE_中国本土版_学习卡.html`：两张端到端验收样例卡（均已通过 `validate_card.py`）。
