一页速查表（闪卡）

一句话定义
FDE = 驻扎客户现场、既写代码又懂业务还扛沟通的"产品落地尖兵"，把通用产品变成客户肯付钱的具体方案，并把一线痛点喂回产品。
本质
编码 + 业务翻译 + 客户沟通 三合一，介于 SWE 与售前之间但更偏技术。
部署闭环
发现需求 → 原型 → 集成客户系统 → 部署上线 → 反馈回产品。
公司视角
缩短销售周期、提高 POC 成功率，ROI 高。
个人视角
杠杆来自"反馈权"与"需求→系统"能力，是职业期权而非当期高薪。
关键风险
易被稀释成出差型集成苦力；体验取决于产品阶段（早期高杠杆 / 成熟低成长）。

题库（考到崩溃）

Q1（初）FDE 是什么的缩写、字面意思？初级
Forward Deployed Engineer，前线部署工程师。
Q2（初）FDE 相比售前(SE) 最关键的不同？初级
FDE 写更多生产级代码、更贴近产品路线图与反馈。
Q3（初）FDE 主要解决公司的什么问题？初级
把通用产品/模型变成客户肯付钱的具体方案，缩短销售周期。
Q4（中）一次部署闭环有哪几步？中级
发现需求 → 原型 → 集成到客户系统 → 部署上线 → 反馈回产品。
Q5（中）FDE 的三项核心能力是什么？中级
编码能力 + 业务/领域翻译 + 客户沟通与信任。
Q6（中）为什么 FDE 体验高度依赖"公司产品阶段"？中级
早期产品=共创高杠杆，成熟产品=实施低成长，同一头衔天差地别。
Q7（高）FDE 真实落地最大的非技术难点通常是什么？高级
无权限 / 无文档 / 客户防备的"政治+环境"夹缝。
Q8（高）什么时候该把问题升级给产品/销售而非自己硬扛？高级
触及产品核心能力缺口、合规/合同风险、超出授权时。
Q9（专）FDE 经历里最该带走的"通用能力"是什么？专家
把模糊需求变成可用系统的能力，产品/创业/架构都通用。
Q10（专）为什么说 FDE 对个人的杠杆来自"反馈权"而非头衔？专家
只有把一线需求接进产品，才同时帮公司省钱、给自己攒稀缺能力；被禁反馈权=两头杠杆都塌。
