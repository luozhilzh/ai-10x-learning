一页速查表（闪卡）

一句话定义
中国 FDE = 在「关系驱动 + 私有化 + 强合规 + 信创」土壤里，把大模型落地成客户肯签单 / 续费的结果的人；比美国多扛「合规 + 关系决策链」两座山。
中美部署形态
中国以私有化 / 一体机 / 信创适配为主；美国以云 API / SaaS 为主——合规与信创驱动。
第四硬技能（合规）
PIPL / 等保三级 / 数据出境安全评估 / 国密 SM 系列 / 算法备案——美国 FDE 在国内基本不直面这套。
discovery 重心
从「用户痛点」转向「决策链 + 关键人 + 数据出域限制」（关系驱动 + 招投标 + 一把手拍板 + 定制化陷阱）。
国内部署闭环
发现决策链 → 私有化 PoC → 信创集成 → 等保上线 → 反馈产品；双向反馈是价值来源。
上场前清单
能讲 3 个不同 / 画得出闭环（含等保 / 数据出域）/ 四项打分知短板 / 按部署形态筛好公司 / 有本地 mini 原型 / 想清反馈权 / 会反向提问。

题库（考到崩溃）

Q1（初）国内 FDE 是什么缩写、字面意思？初级
Forward Deployed Engineer，前沿 / 前线部署工程师。
Q2（初）国内哪些大厂在招 FDE？举 2–3 个。初级
字节（豆包 / 火山）、腾讯云、阿里云、蚂蚁数科、智谱等。
Q3（初）相比传统「实施工程师 / 技术售前」最关键不同？初级
更贴产品、写更多生产级代码、把一线经验反哺成标准资产（而非只交付）。
Q4（中）中美 FDE 部署形态最根本差异？中级
中国以私有化 / 一体机 / 信创适配为主，美国以云 API / SaaS 为主。
Q5（中）为什么说多了一项「第四硬技能」——合规？列举 3 个。中级
PIPL、等保三级、数据出境安全评估（或国密 / 算法备案）。
Q6（中）中国客户文化不同如何影响 discovery？中级
关系驱动 + 招投标 + 一把手拍板 + 定制化陷阱；discovery 重心从「用户痛点」转向「决策链 + 关键人」。
Q7（高）真实落地最大「非技术 + 非纯关系」难点？高级
信创 / 国产芯片与 OS 适配失败、数据不出域致模型效果打折、等保与合规卡脖。
Q8（高）何时升级给产品 / 销售 / 法务而非硬扛？高级
触及合规红线（等保 / 数据出境）、产品核心能力缺口、超授权、招投标违规风险。
Q9（专）为什么说「定制化」是双刃剑？专家
卖结果需要定制，但也最易锁死成实施苦力、攒不到可迁移能力；解法 = 推动共性标准化。
Q10（专）为什么「把个例沉淀成标准模板」是最稀缺杠杆？专家
国内产品成熟度参差 + 定制陷阱，能把共性抽成公司资产的人既帮公司 scale 又攒稀缺能力——这正是面试官最爱听的「反馈权」故事。
