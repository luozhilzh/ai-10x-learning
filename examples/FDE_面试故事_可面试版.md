# FDE 面试故事（可面试版）

> 来源：第 8 题故事骨架扩写。骨架——非结构化→结构化抽取项目里，分类文档、按类配规则/工具/模型控精度、建行业知识库、反哺技术部固化进产品、跨客户精度稳定、获好评。
> 用法：【】内数字请替换成你真实项目数据；没有就填"量级/区间"，别编精确数面试穿帮。

## 一句话定位（30 秒电梯版）

"我做过最像 FDE 的事，是在一个非结构化数据抽取项目里，既扎进客户现场把文档分门别类定标准，又把踩出来的经验反哺成产品的默认能力，让后面好几家客户直接复用。"

---

## 完整故事（STAR，约 5 分钟）

**Situation（背景）**
客户手里有【待填：如"十几万份"】散落各处的非结构化文档——合同、报表、邮件、内部纪要混在一起，想把它们抽成结构化数据喂给下游系统。但文档类型杂、质量参差，用一套通用规则抽，精度上不来，客户一度觉得"这活干不成"。

**Task（任务）**
我作为现场对接的工程师，要在不推翻现有产品的前提下，把抽取精度做到客户能验收的线，同时不能每家客户都从零再来一遍。

**Action（动作）——这里体现 FDE 内核**
1. **Discovery（先摸清现场，不急着写代码）**：我和客户的业务专家【待填：如"花了一周"】一起把文档按类型、来源、质量做了分类，而不是直接套现成模型。先弄清"现场到底有什么"，再决定"怎么抽"。
2. **工程判断力（按类配不同规则/工具/模型）**：不同类型我上了不同的抽取策略和模型组合，把"精度"从不可控变成可控——哪类用规则引擎、哪类上模型、哪类要人工校验，都按类定。
3. **双向翻译（把客户的话翻成方案）**：客户原来只会说"抽不准"，我帮他们把"准"定义成了可度量的指标和分类标准，双方对齐了什么叫成功。
4. **反馈权（把一次性经验变产品能力）**：我没有只服务这一家。我把分类体系和抽取经验沉淀成【待填：如"行业项目知识库"】，并推动技术部门把其中【待填：如"3 类高频文档"】的抽取规则固化进产品后续版本，做成开箱即用的默认模板。
5. **信任（坏消息早说 + 给备选）**：过程中精度不达标时，我提前把差距、原因和备选方案摆给客户，而不是等到验收爆雷。

**Result（结果）**
- 这家客户的抽取精度从【待填：如"约 70%"】提升到【待填：如"90%+ 的可验收区间"】；
- 固化的默认模板让后续【待填：如"5 家"】新客户开箱即用，部署周期缩短【待填：如"约 40%"】；
- 拿到客户和同事的明确好评，也让我从"服务一家"变成"撬动产品"。

## 我对应了 FDE 的哪些内核（面试时可主动点）

| FDE 内核 | 故事里的证据 |
|---|---|
| 派到现场 / 贴着业务 | 和客户业务专家并肩定分类标准 |
| Discovery | 先分类摸清文档类型，再定方案 |
| 双向翻译 | 把"抽不准"翻译成可度量指标 |
| 反馈权 / 杠杆 | 经验固化进产品默认模板，多家复用 |
| 信任 | 坏消息早说、带备选 |

## 面试官可能追问 + 你的应对

- **"你具体怎么定分类的？"** → 讲和客户的 workshop、按"类型/来源/质量"三维切，举一类例子。
- **"精度怎么衡量的？"** → 讲定义了哪些字段的抽取准确率/召回率，抽评标准。
- **"你一个人做的还是团队？"** → 诚实说角色：你主导分类与反哺，技术部配合固化；突出"推动"而非"独吞"。
- **"换家公司没这产品呢？"** → 说内核可迁移：发现场→定标准→沉淀复用的方法，不绑定某产品。
